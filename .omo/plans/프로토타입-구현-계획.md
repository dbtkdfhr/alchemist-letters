## 1. 기술 스택 & 선택 이유

| 기술 | 버전 | 선택 이유 |
|------|------|-----------|
| **React** | 18+ | 컴포넌트 기반 UI, 방대한 생태계 |
| **TypeScript** | 5+ | 타입 안전성, 게임 데이터 구조의 견고한 정의 |
| **Vite** | 6+ | 빠른 HMR, 빌드 속도, 개발 경험 |
| **pnpm** | 최신 | 엄격한 의존성 관리, 디스크 효율 |
| **TailwindCSS** | 3+ | 빈티지 종이/편지 스타일을 유틸리티 클래스로 빠르게 구현 |
| **Zustand** | 4+ | **상태 관리** — 가장 가벼운 Flux 구현, 보일러플레이트 최소, localStorage 미들웨어 내장 |
| **React Router** | 6+ | 화면 전환 (선택 사항, 상태 기반 라우팅으로 대체 가능) |

### Zustand 선택 이유

- 기획서 특성상 **4개 화면**(편지함, 편지 읽기, 재료 조합, 답장 쓰기) + **연금술 노트**로 구성이 단순함
- Redux는 과도한 boilerplate, Recoil/Jotai는 여러 개의 atom이 오히려 복잡함
- Zustand는 **store 하나로 게임 상태 전체 관리 가능** + `persist` 미들웨어로 **localStorage 자동 저장**이 단 2줄
- 게임 특성상 복잡한 비동기 흐름이 없으므로 Zustand로 충분

## 3. 상태 관리 설계 (Zustand)

### 3.1 gameStore — 메인 게임 상태 (persist 미들웨어 사용 → localStorage)

```typescript
interface GameState {
  // 진행 상태
  currentChapter: number          // 현재 챕터 (0-indexed)
  currentPhase: 'main' | 'reading' | 'alchemy' | 'reply' | 'result'
  completedChapters: number[]
  
  // 캐릭터 분기
  marcoAffection: number          // 마르코 친밀도 (-100 ~ 100)
  marcoConfidence: number         // 마르코 자신감 (-100 ~ 100)
  teachingStyle: 'strict' | 'kind' | 'neutral'
  pastOpenness: number            // 과거 공개도 (0 ~ 100)
  
  // 편지 관련
  readLetters: string[]           // 읽은 편지 ID 목록
  sentReplies: Record<string, string>  // 보낸 답장 기록 {편지ID: 선택지ID}
  unlockedLetters: string[]       // 수신 가능한 편지 ID
  
  // 게임 메타
  unlockedEndings: string[]       // 해금된 엔딩
  playTime: number                // 총 플레이 시간 (초)
}
```

### 3.2 alchemyStore — 연금술 상태 (persist 미들웨어)

```typescript
interface AlchemyState {
  // 재고
  ingredients: Record<string, number>  // {재료ID: 보유량}
  
  // 발견한 레시피
  discoveredRecipes: Record<string, RecipeEntry>
  // {레시피ID: { name, ingredients, result, grade, discoveredAt }}
  
  // 시도 기록 (중복 실험 방지)
  attemptedCombos: string[]  // 정렬된 재료ID 조합의 해시
}
```

### 3.3 uiStore — UI 상태 (persist 불필요)

```typescript
interface UIState {
  currentView: 'letterbox' | 'letter' | 'alchemy' | 'reply' | 'notebook'
  isTransitioning: boolean
  bgmPlaying: boolean
  currentBgm: string | null
  modalOpen: boolean
  modalContent: string | null
}
```

## 4. 데이터 구조

### Chapter (챕터 데이터)

```typescript
interface Chapter {
  id: string                     // "ch01", "ch02"...
  title: string                  // "첫 편지"
  act: 1 | 2 | 3                // 막
  letter: {
    from: 'marco' | 'other'
    content: string[]            // 문단 배열 (페이지 전환 단위)
    postscript?: string          // 추신
  }
  symptoms?: string              // 손님 증상 설명
  requiredEffect?: string[]      // 필요한 속성
  replyOptions: ReplyOption[]    // 선택지
  result: {
    success: ChapterOutcome
    failure: ChapterOutcome
    disaster?: ChapterOutcome
  }
  unlockConditions?: {
    requiredAffection?: number
    requiredConfidence?: number
  }
}

interface ReplyOption {
  id: string                     // "a", "b", "c"
  text: string                   // 선택지 텍스트
  type: 'strict' | 'kind' | 'risky' | 'neutral'
  effects: {
    affection?: number
    confidence?: number
    pastOpenness?: number
  }
  recipeRequired?: string        // 필요한 레시피 ID (없으면 레시피 불필요)
}

interface ChapterOutcome {
  nextLetterContent: string[]    // 결과 편지 문단 배열
  effects: CharacterEffects      // 캐릭터 스탯 변화
  unlocks?: string[]             // 해금 요소
}
```

### Ingredient (재료)

```typescript
interface Ingredient {
  id: string
  name: string
  description: string
  category: 'plant' | 'mineral' | 'animal' | 'special' | 'solvent'
  mainAttributes: Attribute[]    // 주 속성 (1~2개)
  subAttribute?: Attribute       // 부 속성 (1개)
  rarity: 'common' | 'uncommon' | 'rare'
  unlockChapter: number          // 등장 챕터
  icon: string                   // 아이콘 경로 또는 SVG
}
```

### Recipe (레시피)

```typescript
interface Recipe {
  id: string
  name: string                   // 플레이어가 명명 가능 (초깃값: "???")
  ingredients: string[]          // 재료 ID 배열 (2~3개)
  result: {
    effect: string               // "진정 연고", "화상 치유 물약" 등
    grade: 'perfect' | 'success' | 'minor' | 'failure' | 'disaster'
    description: string
    potency: number              // 효과 강도 (1~10)
  }
  attributeMatch: {
    primary: Attribute[]         // 일치한 주 속성
    secondary?: Attribute[]      // 일치한 부 속성
  }
}
```

## 7. 에셋 확보 계획

### 7.1 폰트 (웹폰트 — 외부 CDN)

| 폰트 | 용도 | 출처 | 라이선스 |
|------|------|------|----------|
| **Nanum Pen Script** | 제자 마르코 편지 본문 | Google Fonts (눈누) | SIL Open Font License |
| **Kalam** | 스승 편지 본문 | Google Fonts | SIL Open Font License |
| **Noto Sans KR** | UI 버튼/인터페이스 | Google Fonts | SIL Open Font License |

→ **비용: 무료**. Google Fonts CDN 링크로 적용.

### 7.2 종이 질감 (CSS로 생성 or 무료 텍스처)

**선택지 A: 순수 CSS** (추천 — 외부 파일 불필요)
- `background: linear-gradient` + `box-shadow`로 종이 느낌 구현
- 그라데이션 + 약간의 노이즈로 충분한 퀄리티
- `radial-gradient`로 오래된 종이 얼룩 효과

**선택지 B: 무료 텍스처** (더 리얼한 질감)
- 출처: [Transparent Textures](https://www.transparenttextures.com/) — 종이 질감 패턴
- 또는 [Pixabay](https://pixabay.com/) — 스캔한 종이 이미지
- `pngwing.com` — 종이 질감 CC0

→ **추천: 선택지 A (순수 CSS)**. 로딩 시간 단축, 외부 의존성 제거.

### 7.3 재료 아이콘 (20종)

**선택지 A: Game-Icons.net** (무료, SVG, 추천)
- [game-icons.net](https://game-icons.net/) — 4000+ 무료 게임 아이콘
- CC BY 3.0 또는 CC0 라이선스
- 잎사귀, 불, 수정, 물방울 등 재료 콘셉트에 맞는 아이콘 선택
- 64x64 or SVG로 다운로드

**선택지 B: React-icons + 직접 스타일링**
- `react-icons` 패키지에서 비슷한 아이콘 선택
- TailwindCSS로 색상/크기 조정

**선택지 C: 직접 SVG 제작** (시간이 걸리지만 완전 자유)
- 간단한 도형 + 색상으로 자체 제작
- 재료 특성을 아이콘으로 표현

→ **추천: 선택지 A (Game-Icons.net)**. 퀄리티 높고, 프로토타입에 적합.

### 7.4 사운드 (BGM 5종 + SFX 9종)

**BGM 출처: Free Music Archive (FMA)** 또는 **Pixabay Music**
- [Pixabay Music](https://pixabay.com/music/) — 무료, 저작권 표시 불필요
- 장르: Ambient, Piano, Lo-fi
- 키워드: "ambient piano", "calm music", "mystery"

**SFX 출처: Freesound.org**
- [Freesound](https://freesound.org/) — 회원가입 후 무료 다운로드
- Creative Commons 0 (CC0) 또는 CC BY 필터
